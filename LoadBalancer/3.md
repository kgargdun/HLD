# **L4 vs L7 Load Balancing.md**

---

## **3.1 What is L4 vs L7**

### L4 Load Balancer (Transport Layer)

* Operates at TCP/UDP level
* Routes based on:

  * IP address
  * Port

---

### L7 Load Balancer (Application Layer)

* Operates at HTTP level
* Routes based on:

  * URL path
  * Headers
  * Cookies

---

## **3.2 L4 Load Balancing**

### How it works:

* Forwards packets without inspecting content

---

### Pros

* Very fast
* Low overhead

---

### Cons

* No smart routing
* No content awareness

---

### Use Cases

* TCP services
* Databases

---

## **3.3 L7 Load Balancing**

### How it works:

* Terminates connection
* Inspects request
* Routes intelligently

---

### Pros

* Smart routing
* Supports advanced features

---

### Cons

* Higher latency
* More CPU usage

---

### Features

#### Path-based routing

```
/api → Service A  
/images → Service B
```

---

#### Header-based routing

* Route based on user/device

---

#### A/B testing

* Split traffic across versions

---

#### SSL Termination

* LB handles HTTPS → backend gets HTTP

---

## **3.4 Key Differences**

| Feature     | L4        | L7            |
| ----------- | --------- | ------------- |
| Layer       | Transport | Application   |
| Speed       | Faster    | Slower        |
| Routing     | IP/Port   | Content-based |
| Flexibility | Low       | High          |
| CPU Usage   | Low       | Higher        |

---

## **3.5 Real-world Tools**

### L7 Load Balancers

* NGINX
* HAProxy

---

### Cloud LBs

* AWS ALB → L7
* AWS NLB → L4

---

## **3.6 When to Use What (Interview)**

### Use L4 when:

* Need ultra-low latency
* Simple routing
* TCP services

---

### Use L7 when:

* Need smart routing
* Microservices
* API gateways

---

## **3.7 Combined Usage (Real Systems)**

```
Client → L7 LB → L4 LB → Services
```

OR

```
CDN → L7 → App servers
```

---

## **Interview Line**

> “I’ll use L7 load balancing for flexibility, unless latency is critical, then I’d consider L4.”

---

## **Common Mistakes**

* Not knowing difference
* Using L4 when routing logic needed
* Ignoring SSL termination

---

## **Golden Rule**

> “L4 = speed, L7 = intelligence.”
