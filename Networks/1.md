# Networking Fundamentals

## 🧠 Core Mental Model
> Network = unreliable pipe between machines

- Data is broken into small chunks → **packets**
- Packets travel independently (can be lost, delayed, reordered)
- Higher layers (TCP, HTTP) fix these issues

---

## 📦 Basic Units of Data

| Term      | Layer        | Meaning |
|----------|-------------|--------|
| Message  | Application | Full data (e.g., HTTP request) |
| Segment  | Transport   | TCP unit |
| Datagram | Transport   | UDP unit |
| Packet   | Network     | IP unit |
| Frame    | Data Link   | Ethernet unit |

👉 Same data, different names at different layers

---

## ⚡ Key Metrics (VERY IMPORTANT)

### 1. Latency
> Time taken for data to travel from source → destination

- Measured in ms
- Includes:
  - propagation delay
  - transmission delay
  - queuing delay

---

### 2. Bandwidth
> Maximum data transfer capacity

- e.g., 100 Mbps
- Theoretical max speed

---

### 3. Throughput
> Actual data transfer rate achieved

- Always ≤ bandwidth
- affected by:
  - congestion
  - packet loss
  - protocol overhead

---

### 🔥 Memory Hook
> Bandwidth = pipe size  
> Latency = pipe length  
> Throughput = actual water flow  

---

## 🔁 Packet Switching vs Circuit Switching

### Packet Switching (used in Internet)
- Data split into packets
- Each packet routed independently
- Efficient, flexible

### Circuit Switching (old systems)
- Dedicated path established
- Reserved bandwidth
- Example: old telephone systems

---

### 🔥 Memory Hook
> Internet = packet switching → no guaranteed path

---

## 📉 Packet Loss & Retransmission

- Packets can be:
  - lost
  - duplicated
  - reordered

👉 Causes:
- congestion
- network failure

👉 Handling:
- TCP → retransmits
- UDP → ignores

---

## ⏱️ Jitter
> Variation in packet arrival time

- Important for:
  - video calls
  - gaming

---

## 🔌 IP Address (Basic)

- Unique identifier of a machine
- Example: `192.168.1.1`

👉 Two versions:
- IPv4 (limited)
- IPv6 (huge space)

---

## 🔄 Ports (Basic)

- Identify application on a machine

| Port | Usage |
|------|------|
| 80   | HTTP |
| 443  | HTTPS |
| 22   | SSH |

👉 IP = machine  
👉 Port = process

## 🔥 Important Interview Insights

- Network is unreliable → always design with:
  - retries
  - timeouts
  - idempotency

- Latency matters more than bandwidth for most systems

- Multiple small calls > one large call (bad for latency)

---

## 🧠 Quick Recap

- Data → packets → travel independently  
- Network is unreliable  
- Latency, bandwidth, throughput are key  
- IP + Port identify communication endpoints
