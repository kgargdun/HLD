# Transport Layer: TCP vs UDP

---

## 🧠 Core Mental Model
> Transport layer = how data is delivered between processes

- Uses **ports** to identify applications
- Runs on top of IP (network layer)

👉 IP = machine  
👉 Port = process  

---

## ⚖️ TCP vs UDP (High-Yield Table)

| Feature        | TCP                          | UDP                     |
|---------------|------------------------------|--------------------------|
| Type          | Connection-oriented          | Connectionless           |
| Reliability   | ✅ Guaranteed                | ❌ No guarantee          |
| Ordering      | ✅ Maintained                | ❌ Not guaranteed        |
| Speed         | Slower                       | Faster                   |
| Overhead      | High                         | Low                      |
| Use cases     | APIs, DB, web                | Streaming, gaming, DNS   |

---

### 🔥 Memory Hook
> TCP = correctness  
> UDP = speed  

---

## 🔌 Ports (Important)

- Identify application on a host

Examples:
- 80 → HTTP  
- 443 → HTTPS  
- 53 → DNS  

---

## 📡 TCP (Transmission Control Protocol)

---

### 🎯 What problem does TCP solve?
- Unreliable network
- Packet loss
- Out-of-order delivery

---

## ⚙️ TCP Working (Step-by-Step)

### 1. Connection Establishment (3-Way Handshake)

```

Client → SYN →
Server → SYN-ACK →
Client → ACK →

```

👉 Connection established

---

### 2. Data Transfer

- Data split into segments
- Each segment has:
  - sequence number
  - checksum

---

### 3. Acknowledgement (ACKs)

- Receiver sends ACK for received data
- Missing ACK → retransmission

---

### 4. Connection Termination

```

FIN → ACK → FIN → ACK

```

---

## 🔁 Reliability Mechanisms (VERY IMPORTANT)

### 1. Sequence Numbers
- Ensure correct ordering

---

### 2. Retransmission
- Lost packets are resent

---

### 3. Acknowledgements
- Confirm delivery

---

### 4. Sliding Window (Flow Control)
- Controls how much data can be sent before ACK

👉 Prevents overwhelming receiver

---

### 5. Congestion Control
- Adjusts sending rate based on network load

👉 Example:
- slow start
- congestion avoidance

---

### 🔥 Memory Hook
> TCP = “send → wait → adjust”

---

## ⚠️ TCP Limitations

- Head-of-line blocking  
- Connection setup overhead  
- Not ideal for real-time systems  

---

## 📡 UDP (User Datagram Protocol)

---

### 🎯 What problem does UDP solve?
- Need for **low latency**
- Can tolerate data loss

---

### ⚙️ How UDP Works

- No connection setup  
- Sends packets directly  
- No ACKs, no retransmission  

---

### 🔥 Memory Hook
> UDP = “send and forget”

---

## ⚖️ TCP vs UDP (When to Use)

| Scenario | Protocol |
|----------|--------|
| API calls | TCP |
| File transfer | TCP |
| Video streaming | UDP |
| Gaming | UDP |
| DNS lookup | UDP |

---

## 🧠 Real-World Mapping

| System | Protocol |
|--------|--------|
| HTTP APIs | TCP |
| WebSockets | TCP |
| Video calls | UDP |
| DNS | UDP |

---

## 🔥 Interview Insights

### 1. Why not UDP everywhere?
- No reliability
- App has to handle everything

---

### 2. Why not TCP everywhere?
- High latency
- Not suitable for real-time

---

### 3. Head-of-Line Blocking
> One lost packet blocks everything after it

👉 Important for HTTP/1.1, solved partially in HTTP/2, avoided in HTTP/3

---

### 4. Retries & Timeouts
- Built on top of TCP in applications
- Still required despite TCP reliability

---

## 🚀 Example: API Call

1. TCP connection established  
2. HTTP request sent  
3. Server responds  
4. TCP ensures ordered delivery  

---

## 🧠 Quick Recap

- TCP → reliable, ordered, slower  
- UDP → fast, unreliable  
- TCP uses handshake, ACKs, retransmission  
- UDP is minimal overhead  

---

## 🔥 One-Liners (Interview Use)

- “TCP provides reliability over an unreliable network”
- “UDP trades reliability for latency”
- “TCP uses congestion control to adapt to network conditions”
- “UDP is preferred in real-time systems where latency matters more than accuracy”
