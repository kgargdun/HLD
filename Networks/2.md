# OSI & TCP/IP Models

---

## 🧠 Core Mental Model
> Each layer solves ONE problem and hides complexity below

- Layers provide **abstraction**
- Each layer only talks to:
  - layer above (interface)
  - layer below (implementation)

---

## 📚 OSI Model (7 Layers)

| Layer | Name         | Responsibility |
|------|-------------|---------------|
| 7    | Application | User-facing protocols (HTTP, DNS) |
| 6    | Presentation| Data format, encryption |
| 5    | Session     | Connection management |
| 4    | Transport   | Reliable delivery (TCP/UDP) |
| 3    | Network     | Routing (IP) |
| 2    | Data Link   | Local delivery (MAC) |
| 1    | Physical    | Bits over wire |

---

## 🌐 TCP/IP Model (Practical Model)

| TCP/IP Layer | Maps to OSI | Responsibility |
|-------------|------------|---------------|
| Application | 5,6,7      | HTTP, DNS, TLS |
| Transport   | 4          | TCP, UDP |
| Internet    | 3          | IP routing |
| Link        | 1,2        | Physical + MAC |

---

### 🔥 Memory Hook
> OSI = theoretical  
> TCP/IP = real world  

---

## 🔗 OSI vs TCP/IP (Quick Mapping)

| OSI | TCP/IP | Example |
|-----|--------|--------|
| Application | Application | HTTP |
| Presentation | Application | TLS |
| Session | Application | WebSocket |
| Transport | Transport | TCP |
| Network | Internet | IP |
| Data Link | Link | Ethernet |
| Physical | Link | Cable |

---

## 📦 Encapsulation (VERY IMPORTANT)

### What happens when data is sent?

1. Application → creates message (HTTP request)
2. Transport → adds TCP header (port, seq no)
3. Network → adds IP header (IP address)
4. Link → adds MAC header
5. Sent as bits

---

### 🔽 Flow (Top → Bottom)

```text
Application:   HTTP Request
Transport:     [TCP Header + Data]
Network:       [IP Header + TCP Data]
Data Link:     [MAC Header + IP Data]
Physical:      Bits on wire
````

---

### 🔼 Reverse at receiver

* Each layer removes its header
* Passes data upward

---

### 🔥 Memory Hook

> “Wrap → Send → Unwrap”

---

## 📡 Responsibilities by Layer (Interview Gold)

### Application Layer

* Defines **what data means**
* Examples:

  * HTTP → web
  * DNS → name resolution

---

### Transport Layer

* Defines **how data is delivered**
* Features:

  * reliability (TCP)
  * speed (UDP)
  * ports

---

### Network Layer

* Defines **where to send data**
* Uses:

  * IP addresses
  * routing

---

### Data Link Layer

* Defines **local delivery**
* Uses:

  * MAC addresses
  * switches

---

### Physical Layer

* Defines **how bits travel**
* Examples:

  * cables
  * radio signals

---

## 🔥 Key Interview Insights

* Layers = **separation of concerns**
* You can change one layer without affecting others

👉 Example:

* Switch HTTP → gRPC (application layer change)
* TCP/IP remains same

---

## ⚠️ Common Confusions

### 1. HTTP vs TCP

* HTTP → application layer
* TCP → transport layer

👉 HTTP uses TCP underneath

---

### 2. TLS location

* OSI → Presentation layer
* TCP/IP → Application layer

---

### 3. WebSocket

* Starts as HTTP, then upgrades
* Still runs over TCP

---

## 🧠 Real-World Mapping

| Concept     | Real System      |
| ----------- | ---------------- |
| Application | APIs (REST/gRPC) |
| Transport   | TCP connections  |
| Network     | Internet routing |
| Link        | Ethernet/WiFi    |

---

## 🚀 Example: Loading a Website

1. DNS resolves domain → IP
2. TCP connection established
3. TLS handshake (HTTPS)
4. HTTP request sent
5. Response received

👉 Each step uses different layers

---

## 🧠 Quick Recap

* OSI = conceptual (7 layers)
* TCP/IP = practical (4 layers)
* Encapsulation = adding headers layer by layer
* Each layer solves one problem
