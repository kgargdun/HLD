## 🧠 Core Mental Models

- Network = unreliable pipe  
- Layers = separation of concerns  
- TCP = reliability  
- UDP = speed  
- DNS = phonebook  
- CDN = bring data closer  

---

# ⚖️ TCP vs UDP

| Feature | TCP | UDP |
|--------|-----|-----|
| Connection | Yes | No |
| Reliability | ✅ | ❌ |
| Ordering | ✅ | ❌ |
| Speed | Slower | Faster |
| Use case | APIs, DB | Streaming, gaming |

---

# 🌐 HTTP Versions

| Version | Key Feature | Problem Solved |
|--------|------------|---------------|
| HTTP/1.1 | Multiple connections | Basic web |
| HTTP/2 | Multiplexing | HOL blocking |
| HTTP/3 | Runs on UDP (QUIC) | TCP limitations |

---

# 🔌 HTTP vs WebSocket

| Feature | HTTP | WebSocket |
|--------|------|----------|
| Type | Request-response | Full duplex |
| Connection | Short-lived | Persistent |
| Use case | APIs | Real-time |

---

# ⚖️ REST vs gRPC vs GraphQL

| Feature | REST | gRPC | GraphQL |
|--------|------|------|--------|
| Format | JSON | Binary | JSON |
| Performance | Medium | High | Medium |
| Flexibility | Low | Low | High |
| Streaming | ❌ | ✅ | Limited |
| Use case | Public APIs | Internal services | UI-driven apps |

---

# 🔁 Polling vs SSE vs WebSocket

| Feature | Polling | SSE | WebSocket |
|--------|--------|-----|----------|
| Direction | Request-based | Server → Client | Bidirectional |
| Latency | High | Medium | Low |
| Use case | Simple apps | Notifications | Chat |

---

# ⚖️ L4 vs L7 Load Balancing

| Feature | L4 | L7 |
|--------|----|----|
| Speed | Fast | Slower |
| Intelligence | Low | High |
| Routing | IP/Port | URL/Header |

---

# 🔁 Sync vs Async

| Feature | Sync | Async |
|--------|------|------|
| Blocking | Yes | No |
| Coupling | Tight | Loose |
| Complexity | Low | High |
