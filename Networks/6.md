# DNS & Internet Routing

---

## 🧠 Core Mental Model
> DNS = phonebook of the internet

- Humans use domain names (google.com)
- Machines use IP addresses

👉 DNS translates domain → IP

---

# 🌍 DNS Resolution (Step-by-Step)

---

## 🎯 What problem?
- Convert domain name → IP address

---

## ⚙️ Full Flow (VERY IMPORTANT)

1. User enters: google.com
2. Browser checks cache
3. OS checks cache
4. Resolver (ISP) is queried
5. Resolver queries:

   → Root server  
   → TLD server (.com)  
   → Authoritative server  

6. IP returned to client
7. Browser makes request

---

### 🔥 Memory Hook
> Browser → OS → Resolver → Root → TLD → Authoritative

---

## 🧠 Types of Queries

### Recursive
- Resolver does all work  
- Client gets final answer  

---

### Iterative
- Each server returns next step  

---

---

# 🗂️ DNS Hierarchy

---

## Root Servers
- Top-level  
- Know TLD servers  

---

## TLD Servers
- .com, .org, .net  
- Know authoritative servers  

---

## Authoritative Servers
- Store actual records  

---

---

# 📦 DNS Records

| Type | Purpose |
|------|--------|
| A | Domain → IPv4 |
| AAAA | Domain → IPv6 |
| CNAME | Alias |
| MX | Mail server |

---

---

# ⚡ DNS Caching (VERY IMPORTANT)

---

## Levels of Cache

| Layer | Example |
|------|--------|
| Browser | Chrome cache |
| OS | Local DNS cache |
| Resolver | ISP cache |

---

---

## TTL (Time To Live)

- Defines how long record is cached  

---

### 🔥 Memory Hook
> DNS = cache-heavy system  

---

---

# 🌍 CDN & Anycast (Important for HLD)

---

## 🎯 CDN (Content Delivery Network)

- Serve content closer to user  

---

## ⚙️ How it works

1. User requests domain  
2. DNS returns nearest CDN IP  
3. Request served from nearby server  

---

---

## 🌐 Anycast

- Same IP across multiple locations  
- Routed to nearest server  

---

### 🔥 Memory Hook
> Same IP, different locations  

---

---

# 🛣️ Internet Routing (Basic)

---

## 🎯 What problem?
- How packets reach destination IP

---

## ⚙️ How it works

- Routers forward packets  
- Based on routing tables  

---

---

## 🔁 Packet Journey

1. Client → local router  
2. ISP network  
3. Internet backbone  
4. Destination network  

---

---

# ⚠️ Important Concepts

---

## Latency Sources

- distance  
- congestion  
- routing path  

---

---

## Failures

- packet loss  
- DNS failure  
- routing issues  

---

---

# 🧠 Real-World Mapping

| Concept | Real System |
|--------|------------|
| DNS | Every API call |
| CDN | Static assets |
| Anycast | Global services |
| Routing | Internet backbone |

---

---

# 🔥 Interview Insights

- DNS lookup happens before every request  
- Caching reduces latency significantly  
- CDN reduces load on origin servers  
- Anycast improves availability and latency  

---

---

# 🚀 Example: Loading a Website

1. DNS resolves domain  
2. CDN IP returned  
3. TCP connection established  
4. HTTPS request sent  

---

---

# 🧠 Quick Recap

- DNS maps domain → IP  
- Uses hierarchy (root → TLD → authoritative)  
- Heavy caching system  
- CDN + Anycast optimize delivery  

---

---

# 🔥 One-Liners

- “DNS is a distributed hierarchical system”
- “Caching is critical for DNS performance”
- “CDNs reduce latency by serving content closer to users”
- “Anycast routes users to nearest available server”
