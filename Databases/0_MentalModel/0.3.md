# 🧠 0.3 DATA ACCESS PATTERNS

---

## 📘 Definition (textbook-style)

**Data access patterns** describe how an application reads and writes data — including the types of queries, frequency, and structure of access.

---

## 💡 Intuition (Why it exists)

Databases are not just about storing data — they are about **how you access it**.

👉 Same data, different access patterns → completely different DB design.

Example:

* “Get user by ID” → simple index
* “Search text” → needs search engine
* “Get all posts from last week” → range query

> If you design schema without knowing access patterns → system will not scale.

---

## 🧩 Core Access Patterns

---

### 1. Point Lookup (Key-based access)

#### 📘 Definition

Fetching a single record using a unique key.

#### 💡 Example

```sql
SELECT * FROM users WHERE user_id = 123;
```

#### ⚡ Characteristics

* Fastest operation
* O(log n) (B+ tree) or O(1) (hash)

#### 🌍 Real-world Mapping

* Redis
* DynamoDB

---

### 2. Range Queries

#### 📘 Definition

Fetching records within a range of values.

#### 💡 Example

```sql
SELECT * FROM orders WHERE created_at BETWEEN t1 AND t2;
```

#### ⚡ Characteristics

* Requires ordered indexes
* Efficient with B+ trees
* Poor with hash-based systems

#### 🌍 Real-world Mapping

* PostgreSQL
* MySQL

---

### 3. Aggregations

#### 📘 Definition

Computing summary values over data.

#### 💡 Example

```sql
SELECT COUNT(*), AVG(price) FROM orders;
```

#### ⚡ Characteristics

* Expensive on large datasets
* Often needs pre-computation

#### 🌍 Real-world Mapping

* BigQuery
* Redshift

---

### 4. Full-Text Search

#### 📘 Definition

Searching text using keywords, ranking, relevance.

#### 💡 Example

* “Find all posts containing ‘database scaling’”

#### ⚡ Characteristics

* Uses inverted index (not B+ tree)
* Supports ranking, fuzzy search

#### 🌍 Real-world Mapping

* Elasticsearch

---

### 5. Write Patterns (IMPORTANT)

---

#### ✍️ a) Random Writes

* Updates anywhere in dataset
* Expensive for disk (seek cost)

---

#### ➕ b) Sequential / Append Writes

* Writes added at end
* Very efficient

#### 🌍 Mapping

* LSM-based systems → Cassandra

---

#### 🔄 c) Upserts (Update + Insert)

* Common in modern systems
* Needs conflict handling

---

## ⚖️ Access Pattern vs DB Choice (CRITICAL)

| Access Pattern   | Best Choice     |
| ---------------- | --------------- |
| Key lookup       | Redis, DynamoDB |
| Range queries    | PostgreSQL      |
| Aggregations     | BigQuery        |
| Full-text search | Elasticsearch   |
| Heavy writes     | Cassandra       |

---

## 🎯 When to use vs not use

---

### Use key-value DB when:

* Mostly point lookups
* No complex queries needed

---

### Avoid key-value DB when:

* Need range queries
* Need joins or aggregations

---

### Use relational DB when:

* Mixed queries (point + range)
* Need flexibility

---

### Use search engine when:

* Text search required
* Ranking / fuzzy matching needed

---

## 🗣️ Interview One-liner

> “Database design should be driven by access patterns — point lookups, range queries, and aggregations require fundamentally different storage and indexing strategies.”

---

## 🚨 Red Flags / Mistakes

* ❌ Designing schema before knowing queries
* ❌ Using Redis for range queries
* ❌ Using PostgreSQL for full-text search at scale
* ❌ Ignoring write pattern (random vs sequential)
* ❌ Overusing aggregations on OLTP systems
