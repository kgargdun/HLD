# API Design at Scale

## 1. Performance

### Caching

#### Levels

* **Client-side cache**
* **CDN cache**
* **Server-side cache (Redis, in-memory)**

#### What to Cache

* Frequently accessed, read-heavy data
* Example: movie listings, product catalogs

#### Cache Strategies

* TTL (Time-To-Live)
* Cache invalidation (hard problem)

---

### Pagination (Critical at Scale)

* Never return unbounded data

#### Types

* **Offset-based**

  * Simple but slow at large offsets

* **Cursor-based (preferred)**

  * Uses pointer (last_id, timestamp)
  * Scales better

---

### Batch APIs

* Combine multiple requests into one

Example:

* Instead of:

  * `/user/1`
  * `/user/1/orders`
* Use:

  * `/user/1?include=orders`

---

## 2. Reliability

### Timeouts

* Prevent requests from hanging indefinitely

---

### Retries

* Retry failed requests (network issues)

#### Problem

* Can cause duplicate operations

---

### Idempotency (Very Important)

#### Solution to retries

* Same request → same result

Example:

* Use **idempotency key** for POST
* Prevent duplicate payments/orders

---

### Circuit Breaker Pattern

* Stop calling failing service
* Fail fast instead of waiting

States:

* Closed → normal
* Open → failing, stop calls
* Half-open → test recovery

---

## 3. Rate Limiting

### Why?

* Prevent abuse
* Protect backend

### Techniques

* Token Bucket
* Leaky Bucket
* Fixed Window

---

## 4. Security

### Authentication

* Who are you?
* Example: JWT tokens

---

### Authorization

* What can you access?
* Role-based access (RBAC)

---

### Input Validation

* Prevent malformed or malicious input
* Avoid SQL injection, etc.

---

## 5. Observability (Core for Scale)

### Logging

* Track requests, errors

### Metrics

* Latency, error rate, throughput

### Tracing

* Track request across services

---

## 6. API Gateway (Entry Point)

### Responsibilities

* Routing
* Authentication
* Rate limiting
* Logging

---

## 7. Common Interview Mistakes

* Ignoring caching strategy
* Using offset pagination at scale
* Not handling retries/idempotency
* No rate limiting
* Missing timeouts

---

## TL;DR

* Cache aggressively for reads
* Use **cursor pagination**
* Design for **retries + idempotency**
* Protect system with **rate limiting**
* Add **timeouts + circuit breakers**
